Privacy Policy

Last updated: May 30, 2026

Custom Cursor ("the Extension") is a browser extension that lets you apply custom cursor styles to web pages. This policy explains what data the Extension does — and does not — collect.

Data Collection

The Extension collects no personal data, no usage statistics, no analytics, and no browsing history. It never transmits any information to a server, API, or third party.

Permissions & How They Are Used

  • storage — Saves only your chosen cursor preference (a simple string like "heart" or "default") in your browser's local storage. This never leaves your device.
  • activeTab — Lets the Extension read and modify the currently active tab only when you click the Extension's toolbar icon. It has no persistent access to any tab.
  • scripting — Injects a CSS style element into the active tab to apply the cursor you selected. No page content is read, stored, or transmitted.
  • <all_urls> host permission — Required so the custom cursor can appear on any website you visit. The Extension never reads, logs, or sends any page data.

No Network Requests

The Extension makes zero network requests. All cursor graphics are embedded as inline SVG data URIs in the source code — no images, scripts, or fonts are loaded from the internet.

No Cookies

The Extension does not create, read, or use cookies.

No User Accounts

The Extension has no login, no sign-up, and no forms. There is nothing to submit.

Third-Party Services

The Extension does not integrate any third-party analytics, advertising, or tracking services.

Changes to This Policy

If this policy is updated, the "Last updated" date at the top of this page will change. Continued use of the Extension after an update means you accept the revised policy.

Contact

If you have questions about this privacy policy, please open an issue on the project's repository or contact the developer at the email address listed on the Chrome Web Store listing.
